# 个人多端同步记账软件 PRD

## 1. 产品概述

本产品是一款面向个人与家庭用户的日常记账软件，帮助用户快速记录收入和支出，查看资金流向，管理预算，并在手机端与 Web/桌面端之间保持数据同步。

首版产品聚焦“随手记账、清晰统计、多端可用”的核心体验，不覆盖企业财务、专业会计、投资交易或自动银行流水导入等复杂场景。

## 2. 目标用户与使用场景

### 2.1 目标用户

- 有日常收支管理需求的个人用户。
- 希望了解家庭消费结构的家庭用户。
- 希望通过预算控制消费的人群。
- 需要在手机与电脑之间查看、补录或整理账单的用户。

### 2.2 典型场景

- 用户在消费后通过手机快速记录一笔支出。
- 用户月底查看本月总支出、分类占比和账户余额变化。
- 用户在电脑端批量整理账单、筛选历史记录。
- 用户设置月度预算，并在预算接近上限时获得提示。
- 用户离线时先记账，联网后自动同步到其它设备。

## 3. 产品目标

- 降低记账成本，让用户能在较短时间内完成一笔收支记录。
- 帮助用户直观看到收入、支出、结余和消费分类结构。
- 支持多端账号登录，保证同一用户在不同设备上看到一致数据。
- 提供基础预算能力，帮助用户控制月度消费。
- 提供基础数据备份与恢复能力，降低用户数据丢失风险。

## 4. 功能范围与优先级

| 优先级 | 功能 | 说明 |
| --- | --- | --- |
| P0 | 收入/支出记录 | 首版核心功能，支持金额、类型、分类、账户、时间、备注等字段 |
| P0 | 账单列表与筛选 | 支持按时间、类型、分类、账户查看和筛选账单 |
| P0 | 多端账号登录与数据同步 | 同一账号在手机端与 Web/桌面端共享数据 |
| P0 | 月度统计 | 展示月度收入、支出、结余和分类占比 |
| P1 | 分类管理 | 支持默认分类和用户自定义分类 |
| P1 | 账户管理 | 支持现金、银行卡、电子钱包等账户类型 |
| P1 | 预算管理 | 支持设置月度总预算和分类预算 |
| P1 | 年度统计 | 展示年度趋势、月度对比和分类汇总 |
| P1 | 数据备份与恢复 | 支持用户手动备份和恢复个人账本数据 |
| P2 | 提醒能力 | 支持预算超限提醒、每日记账提醒等基础提醒 |

## 5. 核心功能需求

### 5.1 收入/支出记录

- 用户可以新增收入或支出记录。
- 每笔记录至少包含金额、收支类型、分类、账户、发生时间。
- 用户可以选填备注。
- 用户可以编辑或删除自己创建的记录。
- 删除记录前应有确认提示，避免误删。
- 金额必须大于 0，时间不能为空。

### 5.2 分类管理

- 系统提供默认收入分类与支出分类。
- 用户可以新增、编辑、停用自定义分类。
- 已有关联账单的分类不应被直接物理删除，可通过停用隐藏。
- 账单新增时应优先展示常用分类。

### 5.3 账户管理

- 用户可以创建和管理多个资金账户。
- 账户类型包括现金、银行卡、电子钱包和其它。
- 账单记录需要绑定一个账户。
- 系统应根据账单自动计算账户余额变化。
- 用户可以停用不再使用的账户，停用账户不影响历史账单展示。

### 5.4 预算管理

- 用户可以设置月度总预算。
- 用户可以设置支出分类预算。
- 系统应展示预算已使用金额、剩余金额和使用进度。
- 当预算接近或超过上限时，系统应给出提示。

### 5.5 账单列表与筛选

- 用户可以按天、月、年查看账单。
- 用户可以按收入/支出、分类、账户、时间范围筛选账单。
- 用户可以通过关键词搜索备注内容。
- 列表应展示金额、分类、账户、时间和备注摘要。
- 用户可以从列表进入账单详情并进行编辑。

### 5.6 统计报表

- 月度统计应展示总收入、总支出、结余和支出分类占比。
- 年度统计应展示每月收入、支出和结余趋势。
- 统计结果应随账单新增、编辑、删除实时更新。
- 用户可以切换统计时间范围。
- 图表应服务于理解账务情况，不要求首版提供复杂自定义分析。

### 5.7 多端账号登录与数据同步

- 用户可以通过同一账号在手机端与 Web/桌面端登录。
- 同一账号下的账单、分类、账户、预算数据应在多端保持一致。
- 用户离线时可以继续新增、编辑账单。
- 设备恢复联网后，系统应自动同步离线期间产生的数据。
- 普通冲突以最后更新时间为准进行处理。
- 如果同一条数据在多个设备上发生明显冲突，应尽量保留可恢复记录或提示用户确认。

### 5.8 数据备份与恢复

- 用户可以手动触发数据备份。
- 用户可以从已有备份中恢复数据。
- 恢复数据前应提示可能影响当前数据。
- 备份内容至少包含账单、分类、账户和预算。

## 6. 页面/模块说明

### 6.1 首页

- 展示本月收入、本月支出、本月结余。
- 展示预算使用进度。
- 提供快速记账入口。
- 展示近期账单列表。

### 6.2 记账页

- 支持选择收入或支出。
- 支持输入金额、选择分类、选择账户、选择时间、填写备注。
- 支持保存后返回首页或继续记下一笔。

### 6.3 账单页

- 展示账单列表。
- 支持筛选、搜索和切换时间范围。
- 支持查看账单详情、编辑账单、删除账单。

### 6.4 统计页

- 展示月度统计和年度统计。
- 支持查看分类占比、收支趋势和结余趋势。
- 支持切换统计月份或年份。

### 6.5 预算页

- 展示月度总预算和分类预算。
- 支持新增、编辑、删除预算配置。
- 展示预算使用状态和超限提示。

### 6.6 设置页

- 支持账号信息查看。
- 支持分类管理、账户管理。
- 支持数据同步状态查看。
- 支持数据备份与恢复入口。

## 7. 数据需求

### 7.1 核心数据对象

| 数据对象 | 关键字段 |
| --- | --- |
| 用户 | 用户 ID、登录账号、昵称、创建时间 |
| 账单 | 账单 ID、用户 ID、金额、收支类型、分类 ID、账户 ID、发生时间、备注、创建时间、更新时间 |
| 分类 | 分类 ID、用户 ID、名称、收支类型、是否默认、是否停用、排序值 |
| 账户 | 账户 ID、用户 ID、名称、账户类型、初始余额、当前余额、是否停用 |
| 预算 | 预算 ID、用户 ID、预算月份、预算类型、分类 ID、预算金额 |
| 同步记录 | 数据 ID、数据类型、设备 ID、更新时间、同步状态 |

### 7.2 数据规则

- 所有个人账务数据必须归属于明确用户。
- 金额字段应使用固定精度，避免浮点误差。
- 历史账单应保留原始发生时间。
- 分类、账户停用后不影响历史账单统计。
- 多端同步需要记录更新时间，用于判断数据新旧。

## 8. 非功能需求

### 8.1 易用性

- 新增一笔普通账单的流程应尽量简短。
- 常用分类、账户应便于快速选择。
- 首页信息应简洁，优先展示用户最关心的本月收支情况。

### 8.2 性能

- 常规账单列表加载应保持流畅。
- 月度统计结果应在用户切换月份后及时展示。
- 离线数据同步不应明显阻塞用户继续使用。

### 8.3 安全与隐私

- 用户数据应按账号隔离。
- 登录态应有基础安全保护。
- 备份与同步数据应避免明文暴露敏感信息。
- 未经用户授权，不应将个人账务数据用于其它用途。

### 8.4 可靠性

- 离线记账数据应保存在本地，避免应用关闭后丢失。
- 同步失败时应提示用户，并在网络恢复后重试。
- 数据恢复前应提供明确确认，避免覆盖误操作。

## 9. 验收标准

- 用户可以完成新增、编辑、删除收入/支出记录。
- 用户可以按月查看账单列表，并通过分类、账户、收支类型进行筛选。
- 用户可以查看本月收入、本月支出、本月结余和分类占比。
- 用户可以创建自定义分类和账户，并在记账时使用。
- 用户可以设置月度预算，并看到预算使用进度。
- 用户可以在至少两个端登录同一账号，并看到同步后的账单数据。
- 用户离线新增账单后，联网时可以自动同步到其它设备。
- 用户可以手动备份数据，并从备份中恢复。
- 首版产品不出现企业会计、投资交易、银行自动导入等超出范围的功能入口。

## 10. 暂不包含范围

- 企业会计凭证、科目体系和财务报表。
- 发票识别、OCR 票据录入。
- 股票、基金、债券等投资理财交易管理。
- 多人协作账本、家庭成员权限管理。
- 银行卡、支付平台流水自动导入。
- 复杂税务、报销、审批流程。
